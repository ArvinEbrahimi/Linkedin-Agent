# About Section Optimization

Rewrite the LinkedIn **About** section using Problem → Proof → CTA structure.

## Structure (mandatory)
1. **Problem** — Who you help and what pain you solve (2-3 lines)
2. **Proof** — Quantified achievements, credentials, social proof (3-5 lines)
3. **CTA** — Clear next step (connect, DM, visit portfolio, open to roles)

## Requirements
- Scannable with line breaks
- First 2 lines must hook (visible before "see more")
- Natural Persian + technical English if fa-en mode
- No generic "passionate about technology" filler
- Include about_structure field explaining how each part maps

## Length
- 1,500–2,600 characters ideal
